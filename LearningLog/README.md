# 學習日誌系統使用說明

> 記錄 MyBoards 專案的學習過程、遇到的問題與解決方案

---

## 📖 系統說明

這個學習日誌系統幫助你：
- ✅ 記錄每個問題的完整脈絡
- ✅ 追蹤解決過程
- ✅ 累積可複用的知識
- ✅ 快速查找過去的問題

---

## 📁 資料夾結構

```
LearningLog/
├── README.md              # 你正在看的這個檔案
├── INDEX.md               # 所有問題的時間索引
├── TOPICS.md              # 按主題分類的索引
├── QUICK-REF.md           # 常見問題快速參考
├── TEMPLATES/             # 模板資料夾
│   ├── problem.md         # 問題記錄模板
│   └── learning.md        # 學習筆記模板
└── 2026/                  # 按年份分類
    └── 03-March/          # 按月份分類
        └── *.md           # 日誌檔案
```

---

## 🚀 如何記錄新問題

### 方法 1：使用模板（推薦）

1. 複製 `TEMPLATES/problem.md`
2. 重新命名為 `YYYY-MM-DD-問題簡述.md`
3. 放到對應的年月資料夾
4. 填寫內容
5. 更新 `INDEX.md`

### 方法 2：快速記錄

直接在對應月份資料夾建立新檔案，使用以下格式：

```markdown
# 問題標題

**日期：** YYYY-MM-DD  
**標籤：** #標籤1 #標籤2  
**狀態：** 🔴 未解決 / 🟡 進行中 / 🟢 已解決

## 問題描述
...

## 解決方案
...
```

## 🏷️ 標籤系統

使用標籤來分類問題，方便搜尋：

### 技術領域
- `#entity-framework` - EF Core 相關
- `#dependency-injection` - DI 相關
- `#csharp` - C# 語言
- `#aspnet-core` - ASP.NET Core
- `#database` - 資料庫相關
- `#linq` - LINQ 查詢

### 問題類型
- `#bug` - 程式錯誤
- `#concept` - 概念理解
- `#configuration` - 設定問題
- `#performance` - 效能問題
- `#best-practice` - 最佳實踐

### 嚴重程度
- `#critical` - 嚴重：阻擋開發
- `#major` - 重要：影響功能
- `#minor` - 次要：小問題
- `#info` - 資訊：學習筆記

---

## 📊 狀態標記

- 🔴 **未解決** - 問題還沒找到解決方案
- 🟡 **進行中** - 正在嘗試解決
- 🟢 **已解決** - 問題已完全解決
- 🔵 **待驗證** - 解決方案需要進一步驗證
- ⚪ **參考** - 純學習筆記，沒有問題

---

## 🔍 如何查找問題

### 按時間查找
查看 `INDEX.md`，按時間倒序排列

### 按主題查找
查看 `TOPICS.md`，按技術主題分類

### 按關鍵字查找
使用 VS Code 的搜尋功能（Ctrl+Shift+F）搜尋整個 LearningLog 資料夾

### 快速參考
查看 `QUICK-REF.md` 的常見問題解決方案

## 📝 檔案命名規則

```
格式：YYYY-MM-DD-簡短描述.md

範例：
✅ 2026-03-12-dbcontext-constructor-issue.md
✅ 2026-03-15-migration-add-column-error.md
✅ 2026-03-20-understand-linq-joins.md

❌ 避免：
- 問題.md （沒有日期）
- 2026312.md （日期格式錯誤）
- DbContext問題.md （沒有日期，中英混雜）
```
